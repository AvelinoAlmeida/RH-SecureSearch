SSE (Protótipo)

Pesquisa por palavras-chave sobre documentos cifrados. O servidor é considerado não confiável e nunca recebe chaves.

Abrir Protótipo (Demo) /client /metrics /stats

O que o protótipo faz

Lado do Cliente
  • Gera uma chave mestra e deriva chaves para trapdoors e cifra.
  • Cifra documentos com AES-GCM (salário incluído no conteúdo cifrado).
  • Constrói um índice invertido trapdoor → [docIds].
  • Para pesquisar, gera trapdoor com HMAC-SHA256 e envia ao servidor.
  • Recebe IDs/ciphertexts e descifra localmente.
Lado do Servidor (não confiável)
  • Armazena apenas docs cifrados e o índice por trapdoors.
  • Ao receber um trapdoor, devolve docIds (lookup no índice).
  • Não possui chaves, não descifra, não vê salários.
  • Regista leakage: nº de pesquisas por trapdoor.
Demo rápida (30–60s)
Cliente
AES-GCM + HMAC
Cifra + Trapdoor + Descifra
/upload
Servidor
trapdoor → docIds

IDs (+ ciphertext)
Descifra no cliente
Comandos (SSE)
# 1) Build + Upload (cifra + indexa + envia)
python client.py build-and-upload docs/employee_salary_dataset.csv

# 2) Pesquisa cifrada
python client.py search "Marketing"
python client.py search "Porto"
Repete o mesmo termo 2–3 vezes e vê a contagem em /stats (leakage).
Baseline (em claro)
# Pesquisa em claro no CSV (baseline)
python client.py search-plain docs/employee_salary_dataset.csv "Marketing"
Depois abre /metrics para comparar SSE vs plain e ver percentis.
Roteiro (o que dizer em 30s):
1) “Ciframos e indexamos no cliente; o servidor guarda só ciphertexts e um índice por trapdoor.”
2) “Na pesquisa, o servidor recebe trapdoor e devolve IDs; a descifra acontece no cliente.”
3) “O servidor observa repetição de trapdoors (leakage) e nós medimos search/fetch/decrypt e comparamos com plain.”
O que o servidor NÃO aprende
  • Conteúdo dos documentos (salários, etc.).
  • Palavras pesquisadas (“Marketing”, “Lisboa”…).
  • Chaves de cifra/descifra.
Leakage observado (assumido)
  • Padrão: repetição do mesmo trapdoor.
  • Frequência: nº de pesquisas por trapdoor.
  • Tamanho: nº de docIds devolvidos.
Isto é inerente a SSE básico — não é bug.
Frase para fechar
“O servidor mantém-se cego ao conteúdo e aos termos, mas observamos leakage de padrão; o protótipo mede o custo real dessa proteção.”
